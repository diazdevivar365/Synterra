# Onboarding

The canonical onboarding flow lives in [`PLAN.md §D`](../PLAN.md) (Workspace Lifecycle &
Onboarding). The target is **90 seconds from landing page to first brand-intelligence result**:
signup → workspace provisioning → magic-link sign-in → first research run kick-off happens in one
guided swim-lane, with the Aquila call fired in parallel with the welcome email so the user lands in
an already-useful workspace.
